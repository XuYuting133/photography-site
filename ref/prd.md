# 摄影师个人作品集网站 · 产品需求文档 (PRD)

> **版本**：v1.0  
> **更新日期**：2026-07-12  
> **项目类型**：个人摄影师作品集展示网站

---

## 1. 项目背景

作为一名摄影师，你需要一个属于自己的线上作品展示空间。它不应受限于第三方平台的模板和规则，而应完全服务于你的作品和审美。你选择通过 VibeCode（AI 辅助编程）的方式自行构建，既能实现高度定制化，又能以极低的成本快速上线。

---

## 2. 项目目标

- **展示作品**：以最纯粹、最沉浸的方式呈现街拍与风光摄影作品。
- **建立品牌**：通过独特的视觉风格和内容编排，传递个人摄影美学。
- **建立联系**：为潜在客户、合作方或同好提供一个简单直接的沟通入口。
- **极低成本**：利用 Vercel 等免费或低成本的现代工具，将年度运维费用控制在域名费用以内。
- **极简维护**：作品更新流程简单高效，无需登录后台，修改代码即完成更新。

---

## 3. 目标用户

| 用户类型 | 描述 |
| :--- | :--- |
| **核心用户** | 摄影师本人（你），负责内容更新和网站维护。 |
| **访客** | 潜在客户、策展人、摄影同好、朋友及家人。他们通过网站浏览作品、了解摄影师并建立联系。 |

---

## 4. 用户故事

- 作为**访客**，我希望打开网站时能立即看到高质量的摄影作品，而非复杂的文字或动画。
- 作为**访客**，我希望浏览作品时体验流畅，图片加载迅速，布局在不同设备上都能良好展示。
- 作为**访客**，我希望了解摄影师的艺术理念和背景故事。
- 作为**访客**，我希望能够方便地找到联系摄影师的方式。
- 作为**摄影师**，我希望更新作品集时操作简单，无需处理复杂的后台或数据库。
- 作为**摄影师**，我希望网站的设计风格能匹配街拍/风光作品的调性。

---

## 5. 功能需求

### 5.1 作品展示（核心功能）
- **作品网格**：首页/作品页以网格形式展示所有摄影作品缩略图。
- **图片浏览**：点击缩略图可查看大图（如 Lightbox 弹窗或独立详情页）。
- **作品信息**：每张作品可展示标题、拍摄日期、拍摄地点、简短描述等元数据。
- **分类/筛选**（可选）：可按“街拍”、“风光”等分类筛选作品。
- **项目分组**（可选）：可按拍摄项目或旅行目的地对作品进行分组展示。

### 5.2 关于我
- 展示摄影师的个人简介、艺术理念、创作经历。
- 可包含摄影师肖像、个人风格陈述、展览或出版经历等。

### 5.3 联系我
- 提供一个联系表单，访客可填写姓名、邮箱和留言内容。
- 表单提交后，内容通过第三方服务（如 Formspree）推送至摄影师邮箱，无需后端存储。
- 可展示社交媒体链接（如 Instagram、微博等）。

### 5.4 首页（可选）
- 精选几张代表作作为视觉引入。
- 简洁的导航，引导访客进入“作品”或“关于”页面。

## 6. 非功能需求

| 需求 | 说明 |
| :--- | :--- |
| **性能** | 页面加载迅速，图片利用 CDN 加速，首屏加载时间控制在 2 秒以内。 |
| **响应式** | 完美适配桌面端、平板和移动端设备。 |
| **可访问性** | 遵循基本的 Web 可访问性标准（如语义化 HTML、合理的颜色对比度）。 |
| **安全性** | 通过 Vercel 自动启用 HTTPS，确保网站数据传输安全。 |
| **可维护性** | 代码结构清晰，作品数据集中管理（`works.js`），便于日常更新。 |

## 7. 设计方向

设计方向已在 `design.md` 中详细定义，核心原则摘要如下：

- **作品至上**：一切设计服务于作品的展示。
- **极简主义**：大量留白、精简元素、减少干扰。
- **叙事性**：引导访客以特定节奏浏览作品，创造沉浸式体验。
- **色彩策略**：采用中性背景衬托作品。街拍推荐**暗调风格**，风光推荐**明亮通透**风格。

---

## 8. 技术约束

技术选型详见 `tech-stack.md`，核心约束如下：

| 约束 | 说明 |
| :--- | :--- |
| **零数据库** | 不使用任何关系型数据库或 NoSQL 数据库，作品数据以 JavaScript 数组硬编码。 |
| **纯静态托管** | 网站为纯静态页面，不依赖服务端运行时。部署于 Vercel，利用其全球 CDN 加速。 |
| **图片分离** | 图片不存放在代码仓库中，而是使用 Vercel Blob 或其它云存储服务，实现图片与代码分离。 |
| **前端框架** | 使用 React 或 Vue（二选一），配合 Vite 构建工具。 |
